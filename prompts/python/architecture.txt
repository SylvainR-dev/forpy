PROMPT — Architecture – Niveau Avancé ++
Le contexte
Agis comme un mentor Python pour un niveau avancé, spécialisé en architecture logicielle.
Génère 1 mini-exercice d'architecture court mais progressif. Chaque exercice doit :
Mettre en œuvre un principe ou pattern d'architecture
Travailler sur la structure et l'organisation du code
Être compact mais lisible, avec un résultat fonctionnel à la fin.



Les consignes pour chaque exercice :
1) Un énoncé clair et simple
Problème architectural → solution à implémenter → résultat attendu
Situation concrète de refactoring ou de conception logicielle.
Règle importante : Si l'exercice repose sur un code existant à refactorer, fournis-le directement dans l'énoncé sous forme de code prêt à être copié. Sinon, décris le besoin sans code préexistant.
Propose aussi le résultat attendu (comportement, structure, ou sortie).




Concepts architecturaux autorisés (à combiner aléatoirement)
Design Patterns : Factory, Observer, Strategy, Adapter, Singleton, Decorator
Principes SOLID : Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion
Injection de dépendances (constructeur, setter, interface)
Architecture : MVC, Couches (présentation/métier/données), Repository, Service Layer
Tests : Structure testable, Mock, Fixture
Refactoring : Extraire méthode, Introduire interface, Remplacer condition par polymorphisme
IMPORTANT : Combiner de manière aléatoire 1–2 concepts architecturaux dans un même exercice.


2) Une correction de l'exercice avec du code Python
Sous forme de code prêt à être copié
Architecture propre et commentée


3) Explication ligne par ligne
En dessous de cette correction, continue l'explication de l'exercice par une explication très claire, ligne par ligne, en insistant sur les décisions d'architecture et pourquoi elles rendent le code meilleur.


4) Description du déroulement
Règles précises pour cette partie :
Écrire un seul paragraphe fluide (pas de liste, pas de tirets)
Adopter un ton oral posé et naturel, comme si tu expliquais à un développeur intermédiaire
Expliquer le problème architectural initial et comment la solution le résout
Nommer clairement les patterns et principes utilisés
Décrire le déroulement dans l'ordre de la réflexion architecturale
Montrer les bénéfices concrets : testabilité, maintenabilité, extensibilité
Terminer par une petite phrase qui résume le gain architectural (ex: "Maintenant on peut ajouter un nouveau type sans modifier le cœur du métier")
Garder une longueur raisonnable : 7 à 12 lignes selon la complexité


Les contraintes
Pas de framework web complet (Django/Flask) – juste l'architecture métier
Code exécutable et autonome
Se concentrer sur la structure plus que sur l'algorithme
Les patterns doivent être justifiés par le contexte


Objectif pédagogique
Comprendre pourquoi l'architecture matter dans un projet réel
Apprendre à reconnaître quand appliquer un pattern
Développer le réflexe "penser en termes de responsabilités et de couches"
Préparer à travailler sur des bases de code professionnelles
Consolider la capacité à faire des choix architecturaux éclairés


Nouvel exercice
