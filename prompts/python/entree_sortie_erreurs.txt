PROMPT — Exercices Python (Entrée/Sortie et erreurs)
Le contexte
Agis comme un mentor Python spécialisé dans la gestion des fichiers et des erreurs.
Génère 1 mini-exercice qui fait travailler la lecture/écriture de fichiers et la gestion d'exceptions.
Les consignes pour chaque exercice :
1) Un énoncé clair et simple
L'exercice doit combiner 2 à 3 concepts parmi :
Lecture/écriture de fichiers (txt, csv)
Context managers (with open...)
Gestion d'exceptions (try/except/finally/else)
Lever ses propres exceptions (raise)
Validation de données lues
Règle importante : L'énoncé doit simuler un fichier existant (décrire son contenu) mais ne pas fournir le fichier lui-même. L'élève devra le créer ou le simuler pour tester. Exemple de description : "Le fichier notes.csv contient des lignes avec nom et note séparés par une virgule : Alice,15\nBob,8\n..."
Propose aussi le résultat attendu. 



2) Une correction de l'exercice avec du code Python. 
Sous forme de code prêt à être copié


3) Explication ligne par ligne.


En dessous de cette correction, continue l’explication de l'exercice par une explication très basique, ligne par ligne, sans jargon.
Fais en sorte que chaque exercice soit complet et se termine par l’affichage du résultat, de manière à pouvoir l’exécuter directement. 

4) Description du déroulement

Règles précises pour cette partie :

- Écrire **un seul paragraphe fluide** (pas de liste, pas de tirets, pas de découpage ligne par ligne).
- Adopter un ton **oral posé, naturel, comme si tu expliquais le code à voix haute à un élève débutant**.
- Rester **clair, pragmatique et professionnel** — pas de métaphores poétiques forcées (pas de « ruban qui se déchire », « boîte qui crie True », « couloir sombre », etc.).
- Intégrer **explicitement les données des exemples donnés dans l’énoncé** (les valeurs des print() fournis) pour montrer concrètement ce qui se passe à chaque appel.
- Nommer clairement les éléments du code dans le texte : nom des variables, mots-clés, valeurs testées, résultats renvoyés, etc.
- Expliquer le cheminement : tentative d'ouverture → si erreur → si succès → fermeture
- Montrer ce qui se passe dans chaque bloc try/except
- Décrire le déroulement **dans l’ordre d’exécution**, de façon linéaire et logique.
- Terminer par une petite phrase qui résume simplement l’idée générale de la fonction (ex. : « En gros c’est toujours : test → return » ou « Découpage, comptage, renvoi du nombre »).
- Garder une longueur raisonnable : environ 5 à 9 lignes de texte selon la complexité (pas un roman, pas une phrase unique trop dense non plus).




Les contraintes :


Pas de POO
Utilisation obligatoire de with pour les fichiers
Au moins deux types d'exceptions à gérer


Objectif pédagogique
Comprendre que les fichiers peuvent manquer, être corrompus, ou mal formatés, et qu'un programme robuste doit anticiper ces cas.
